# UCAS2026数据结构OJ习题解答
- 懒得写解析
- C和C++代码都有，vector和string太好用了你知道吗
- OJ的样例和输入不太匹配，有些样例输入全是一位整数，但测评机输入可能是char；有些样例是a-z，测评机会给出特殊字符char输入
  - 输入判断的条件放宽点...
  - 个别题目会卡输出，测评机不忽略行末空格和文件末换行，如果感觉自己算法正确但WA不妨看看是不是输出格式的问题
- 各章节对应的内容：
  - 第一章 绪论
  - 第二章 线性表
    - 2.22 输入有特殊字符，不要用a-z和A-Z做输入判断
  - 第三章 栈和队列
    - 3.20 （题意不明）用2替换1即可 **题干已修复**
  - 第四章 串
  - 第五章 数组和广义表
  - 第六章 树和二叉树
    - 6.43 不忽略行末空格，卡输出
    - 6.20 （题意有误）输入是已经转成二叉树的森林，连接若干棵树为一棵二叉树即可；题目输入的节点值就是下标，e.g. 第3行输入应该是第二个节点，那么节点值就是2
      - 不用维护map来做val-idx映射
- 第七章 图论
  - 7.14 评测对行末空格严格，注意输出处理
  - 7.15 IV操作，如果该节点已经存在，那么将后续节点编号顺延一位，插入该节点
